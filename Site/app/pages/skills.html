When I work on a project with C++ I feel like I am taking all of my tools out of the shed, whereas with other languages I feel I'm just approaching the project with a toolbox. Generally this is my prefered language for personal projects. I frequently look up blogs on interesting topics and libraries using C++ as this is a language I love to explore. Additionally I have been following the C++ standarization process quite closely over the last couple of years and can't wait to start see the C++14 and C++17 features as they roll out over the next few years.

Additionally I have experience working with pure C on Linux machines (specifically Ubuntu), Windows running with Cygwin and on an embeded system called the ColdFire. Futhermore my Windows 8 projects have dealt with Microsofts C++/CX extensions.

Haxe, Javascript, MATLAB

Haxe, Javascript and MATLAB are other languages I've used but do not consider myself experienced enough with to say that I am proficient.

Haxe: I was exploring this language as an alternative to AS3 by using OpenFL. Haxe's ability to compile into several other languages such as AS3, C++, C# and javascript seems to make it quite flexible and I expect it would be fairly simple to find a means to integrate Haxe into an existing system. What interested me most was that it had better language features than AS3, in particular it was statically typed, but could easily be converted into AS3 allowing it to be compiled into a SWF. That SWF could then easily be consumed by either Adobe Air or Scaleform which means that switching to using Haxe instead of AS3 would not require much retooling work of an existing production pipeline. In the future I really want to sit down and work on a small game or other application using Haxe and either OpenFL or Flambe.
